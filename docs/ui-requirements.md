# UI 필수 조건 및 개선 포인트

이 문서는 사용자가 지시한 UI 필수 조건과 개선 사항을 기록합니다.

---

## 완료된 필수 조건

### 1. 페이지 상태 유지 (Session Persistence)
- **요구**: 탭 이동 시 이전 질의/결과가 유지되어야 함
- **원인**: React SPA에서 페이지 이동 시 컴포넌트 언마운트 → state 초기화
- **해결**: `sessionStorage`에 주요 state 저장/복원
- **적용 페이지**: AskPage, SqlPage, RagPage
- **미적용**: ChatPage (대화 히스토리는 백엔드에 저장되므로 불필요)

### 2. 질의 히스토리 보관
- **요구**: GPT처럼 모든 질의 히스토리를 보관
- **해결**: ChatPage — 대화 목록 사이드바 (백엔드 저장)
- **해결**: SqlPage — sqlHistory 배열로 질의/결과 히스토리 유지

### 3. AI 채팅과 업무 질의 분리
- **요구**: AI 채팅은 자유 대화, 업무 질의는 RAG+DB 기반
- **해결**: /chat = 순수 LLM (RAG 없음, 2초 응답), /ask = 멀티에이전트 (RAG+DB)

### 4. 대화 길어질 때 성능
- **요구**: 대화가 길어져도 클라이언트가 버벅이지 않아야 함
- **프론트 해결**: 메시지 윈도잉 (최근 30개만 렌더링, "더 보기" 버튼)
- **백엔드 해결**: 대화 압축 (오래된 메시지 → 요약, 최근 10개만 LLM에 전송)

### 5. KO/EN 다국어
- **요구**: 메뉴명 한글/영어 전환
- **해결**: Sidebar 언어 토글 버튼, navItems에 ko/en 필드

### 6. 에러 메시지 표시
- **요구**: API 200이어도 비즈니스 에러면 내용 표시
- **해결**: text2sql에서 `{"ok": false, "error": "..."}` 반환 → 프론트에서 표시

### 7. 상세 로그
- **요구**: 에러 발생 시 백엔드에서 확실히 구분되게 로깅
- **해결**: [CHAT] [SQL] [RAG] [UPLOAD] [AGENT] [ORCHESTRATOR] [AGENT-TOOL] [AGENT-MGMT] 태그

## 아직 미적용 / 향후 개선

### 8. 가상 스크롤 (Virtual Scroll)
- **현재**: 메시지 30개 윈도잉 ("더 보기" 버튼)
- **이상적**: react-virtuoso로 DOM에 보이는 것만 렌더링
- **우선순위**: 낮음 (현재 방식으로 충분)

### 9. 실시간 에이전트 진행 표시
- **현재**: "에이전트 협업 중..." 로딩만 표시
- **이상적**: "품질 분석관 실행 중... → 문서 검색관 실행 중..." 단계별 표시
- **방법**: SSE(Server-Sent Events)로 에이전트 실행 상태 실시간 전달

### 10. 다크모드
- **현재**: Tailwind CSS 변수로 준비되어 있으나 토글 미구현
- **방법**: `document.documentElement.classList.toggle('dark')` + localStorage 저장

### 11. 반응형 레이아웃
- **현재**: 데스크탑 기준 고정 레이아웃
- **이상적**: 모바일/태블릿 대응

### 12. 업무 질의 히스토리
- **현재**: 탭 이동 시 마지막 질의만 유지
- **이상적**: ChatPage처럼 과거 질의 목록 사이드바

---

## 개선 원칙

1. **상태 유지**: 페이지 이동해도 작업 내용이 사라지면 안 됨
2. **에러 가시성**: 에러가 발생하면 사용자에게 명확히 보여야 함
3. **응답 속도**: 어떤 상황에서도 UI가 버벅이면 안 됨
4. **점진적 로딩**: 대용량 데이터는 페이지네이션 또는 윈도잉으로 처리
